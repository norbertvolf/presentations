Git

Git je distribuovaný systém správy verzí

Dokumentace

Všechny příkazy i popis práce s větvemi je popsána v knize Pro Git, tuto knihu si můžete přečíst i v češtině. Pro práci vývojáře jsou důležité první tři kapitoly. Pokud se chcete dozvědět něco o kořenech Gitu tak mrkněte na tuhle přednášku.

Historie

  • Git vytvořil Linus Torvalds v roce 2005
  • Git vznikl, protože neexistoval nástroj vhodný pro vývoj jádra
  • Nyní ho vyvíjí Junio Hamano, japonský developer
  • Dnes široce používaný včetně největších softwarových firem

Diff & patch


	diff -u original.c new.c > original.patch
	patch original.c < original.patch

Git je distribuovaný

  • Všechny operace nutné pro vývoj probíhají proti lokální databázi
  • Každý repozitář obsahuje kopii historie
  • Není žádný centrální repozitář
  • Není žádné pevně definované workflow
